# 屏幕常亮控制

## 1. 功能概述

在 Agent 流式输出或长时间同步时，防止系统自动熄屏打断用户体验。该功能通过 **Raw JNI** 直接调用 Android `Window.addFlags(FLAG_KEEP_SCREEN_ON)` 和 `clearFlags(...)`，不经过 Kotlin 插件层。

> **设计决策**：仅涉及一次性的 Window flag 操作，使用 Raw JNI 比新增 Kotlin 方法更简洁。参见 `docs/ANDROID_PLUGIN_MANAGEMENT.md` §6.1 三条铁律之第 3 条。

### 3.2 `clear_keep_screen_on`

与 `set_keep_screen_on` 对称，调用 `clearFlags(FLAG_KEEP_SCREEN_ON)` 恢复默认熄屏策略。

## 4. JNI 调用链拆解

```rust
let window = app
    .get_webview_window("main")
    .ok_or("main window not found")?;

window
    .as_ref()
    .with_webview(|webview| {
        webview.jni_handle().exec(move |env, activity, _webview| {
            // 1. 调用 activity.getWindow()
            let window = env
                .call_method(activity, "getWindow", "()Landroid/view/Window;", &[])
                .unwrap(); // 内部处理错误

            // 2. 调用 window.addFlags(FLAG_KEEP_SCREEN_ON)
            env.call_method(
                window.l().unwrap(),
                "addFlags", "(I)V",
                &[JValue::Int(FLAG_KEEP_SCREEN_ON)],
            )
        });
    })
    .map_err(|e| format!("with_webview failed: {:?}", e))?;
```

| 步骤 | JNI 签名 | 说明 |
|------|----------|------|
| `getWindow` | `()Landroid/view/Window;` | 从 Activity 获取 Window 对象 |
| `addFlags` / `clearFlags` | `(I)V` | 接收一个 int 参数，无返回值 |
| `FLAG_KEEP_SCREEN_ON` | `0x00000080` | Android SDK 常量，保持屏幕唤醒 |

---

## 5. 错误处理

| 错误来源 | 错误信息 | 触发条件 |
|----------|----------|----------|
| `get_webview_window` | `"main window not found"` | 名为 `"main"` 的 WebView 窗口不存在 |
| `with_webview` | `"with_webview failed: ..."` | JNI 执行环境异常 |
| `getWindow` | 日志输出 `[VcpMobilePlugin] getWindow failed` | JNI 调用 `getWindow` 返回错误 |
| `addFlags`/`clearFlags` | 日志输出 `[VcpMobilePlugin] addFlags failed: ...` | JNI 调用 addFlags 抛出异常 |

> **注意**：内部 JNI 调用错误仅通过 `log::error!` 记录，不中断 `Result` 返回。也就是说，即使 addFlags 失败，前端收到的仍是 `Ok(())`。这是已知设计（一次性标志操作，失败不影响核心功能）。

## 6. 使用示例

### 前端调用

```typescript
import { setKeepScreenOn, clearKeepScreenOn } from 'tauri-plugin-vcp-mobile-api';

// 开始流式输出前
await setKeepScreenOn();

// 流式输出结束后
await clearKeepScreenOn();
```

### 典型场景

```
用户点击发送消息
    ↓
前端调用 startStreamService(agentName)  [启动保活]
    ↓
前端调用 setKeepScreenOn()               [防止熄屏]
    ↓
... 流式输出中 ...
    ↓
流结束
    ↓
前端调用 stopStreamService()             [停止保活]
    ↓
前端调用 clearKeepScreenOn()             [恢复熄屏]
```

## 7. 与 Kotlin 侧的对照

虽然 `screen.rs` 使用 Raw JNI，但 Kotlin 侧 `VcpMobilePlugin.kt` 也保留了等效方法：

```kotlin
@Command
fun setKeepScreenOn(invoke: Invoke) {
    activity.window.addFlags(android.view.WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON)
    invoke.resolve()
}
```

**当前未使用**：Rust 侧直接走 Raw JNI，Kotlin 侧方法作为备用存在。如果未来需要从前端直接通过 `evaluateJavascript` 触发屏幕控制，可启用 Kotlin 侧方法。
