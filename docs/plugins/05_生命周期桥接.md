# 生命周期桥接

## 1. 功能概述

通过 Android `DefaultLifecycleObserver` 监听 Activity 生命周期变化（resume / pause / stop / config-changed / low-memory），将事件通过 `evaluateJavascript` 直接注入前端 `window.CustomEvent`，使 Vue 层能够响应应用前后台切换、深色模式变化、内存告警等系统级事件。

> **设计决策**：与 `KeyboardInsetsManager` 保持一致，使用 `evaluateJavascript` + `window.addEventListener` 通道，前端无需引入 Tauri 事件 API 即可接收。

## 3. 事件类型与触发时机

| 事件名 | 触发时机 | Payload | 前端典型用途 |
|--------|----------|---------|-------------|
| `vcp-lifecycle` | `onResume` | `{ "state": "resume" }` | 恢复 WebSocket 连接、刷新未读消息 |
| `vcp-lifecycle` | `onPause` | `{ "state": "pause" }` | 暂停动画、保存草稿 |
| `vcp-lifecycle` | `onStop` | `{ "state": "stop" }` | 断开非必要连接、释放资源 |
| `vcp-lifecycle` | `onConfigurationChanged` | `{ "state": "config-changed", "isDarkMode": true/false }` | 响应系统深色模式切换 |
| `vcp-lifecycle` | `onLowMemory` | `{ "state": "low-memory" }` | 清理图片缓存、释放大对象 |

### 3.1 深色模式检测

```kotlin
fun onConfigurationChanged(newConfig: Configuration) {
    val uiMode = newConfig.uiMode and Configuration.UI_MODE_NIGHT_MASK
    val isDark = uiMode == Configuration.UI_MODE_NIGHT_YES
    emit("vcp-lifecycle", mapOf(
        "state" to "config-changed",
        "isDarkMode" to isDark
    ))
}
```

- **`Configuration.UI_MODE_NIGHT_MASK`**：提取夜间模式标志位。
- **`UI_MODE_NIGHT_YES`**：当前处于深色模式。

### 3.2 内存告警

```kotlin
fun onLowMemory() {
    emit("vcp-lifecycle", mapOf("state" to "low-memory"))
}
```

> **注意**：`onLowMemory()` 不是 `DefaultLifecycleObserver` 的标准回调，需要由外部（如 `VcpMobilePlugin` 或 `MainActivity`）在收到系统 `onLowMemory()` 时显式调用。

## 4. 事件格式标准

```kotlin
private fun emit(eventName: String, detail: Map<String, Any?>) {
    val json = serializeValue(detail)
    val script = "window.dispatchEvent(new CustomEvent('$eventName', { detail: $json }))"
    webViewRef?.evaluateJavascript(script, null)
}
```

与 `KeyboardInsetsManager` 共用相同的 `serializeValue` / `escapeJson` 实现，保持序列化行为一致。

## 5. 前端接收方式

```typescript
window.addEventListener('vcp-lifecycle', (e: CustomEvent) => {
    const { state, isDarkMode } = e.detail;
    switch (state) {
        case 'resume':
            // 恢复连接
            break;
        case 'pause':
            // 暂停操作
            break;
        case 'stop':
            // 释放资源
            break;
        case 'config-changed':
            // 切换主题
            toggleTheme(isDarkMode);
            break;
        case 'low-memory':
            // 清理缓存
            clearImageCache();
            break;
    }
});
```

## 6. 生命周期绑定

```kotlin
// VcpMobilePlugin.kt
class VcpMobilePlugin(private val activity: Activity) : Plugin(activity) {
    private val lifecycleBridge = LifecycleBridge()

    override fun load(webView: WebView) {
        super.load(webView)
        lifecycleBridge.attach(activity, webView)
    }

    override fun onConfigurationChanged(newConfig: Configuration) {
        super.onConfigurationChanged(newConfig)
        lifecycleBridge.onConfigurationChanged(newConfig)
    }
}
```

### 6.1 `attach()` 流程

```kotlin
fun attach(activity: Activity, webView: WebView) {
    webViewRef = webView
    if (activity is LifecycleOwner) {
        activity.lifecycle.addObserver(this)
    }
}
```

- **`LifecycleOwner`**：`AppCompatActivity` 实现了此接口。
- **`addObserver(this)`**：将 `LifecycleBridge`（即 `DefaultLifecycleObserver`）注册到 Activity 生命周期。

---

## 7. 与 `KeyboardInsetsManager` 的对比

| 维度 | `KeyboardInsetsManager` | `LifecycleBridge` |
|------|------------------------|-------------------|
| 监听目标 | `WindowInsets` 变化 | `Activity` 生命周期 |
| 触发频率 | 高（键盘动画期间连续触发） | 低（仅在状态切换时触发） |
| 事件名 | `vcp-keyboard-inset` | `vcp-lifecycle` |
| Payload | `height`, `visible`, `safeAreaBottom` | `state`, `isDarkMode` |
| 注册方式 | `ViewCompat.setOnApplyWindowInsetsListener` | `LifecycleOwner.lifecycle.addObserver` |
| 序列化器 | 手写 `serializeValue` | 手写 `serializeValue`（同一套） |

---

## 8. 关键约束

1. **`Plugin.load(webView)` 时机**：`attach()` 必须在 `load()` 中调用，确保 WebView 已初始化。参见 `docs/ANDROID_PLUGIN_MANAGEMENT.md` §7.3：`Plugin.load()` 不可靠，但 `KeyboardInsetsManager` / `LifecycleBridge` 的 `attach()` 放在 `load()` 中是经过验证的可靠做法。

2. **配置变化回收**：`onConfigurationChanged` 不会触发 Activity 重建时（即 `android:configChanges` 包含 `uiMode`），系统直接回调此方法。前端应注意避免在 `config-changed` 事件中进行耗时操作。
