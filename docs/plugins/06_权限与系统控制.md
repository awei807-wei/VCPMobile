# 权限与系统控制

## 1. 功能概述

提供 Android 运行时权限的查询与请求能力，以及系统级控制命令（返回桌面）。涵盖通知、存储、电池优化三项权限，并处理 Android 版本差异（Android 13+ 细分媒体权限 vs 传统存储权限）。

## 3. 权限检查

### 3.1 接口

```rust
#[derive(Serialize, Deserialize)]
pub struct PermissionStatus {
    pub notification: bool,
    pub storage: bool,
    pub battery: bool,
}

#[tauri::command]
pub fn check_all_permissions<R: Runtime>(app: AppHandle<R>) -> Result<PermissionStatus, String>;
```

| 属性 | 说明 |
|------|------|
| 前端命令 | `plugin:vcp-mobile\|check_all_permissions` |
| 参数 | 无 |
| 返回值 | `PermissionStatus` JSON 对象 |
| 桌面端行为 | 始终返回 `{ notification: true, storage: true, battery: true }` |

### 3.2 Kotlin 实现

```kotlin
@Command
fun checkAllPermissions(invoke: Invoke) {
    val pm = activity.getSystemService(Context.POWER_SERVICE) as PowerManager

    val notificationGranted = if (Build.VERSION.SDK_INT >= 33) {
        ContextCompat.checkSelfPermission(activity, android.Manifest.permission.POST_NOTIFICATIONS) == PackageManager.PERMISSION_GRANTED
    } else {
        true
    }

    val storageGranted = if (Build.VERSION.SDK_INT >= 33) {
        ContextCompat.checkSelfPermission(activity, android.Manifest.permission.READ_MEDIA_IMAGES) == PackageManager.PERMISSION_GRANTED
    } else {
        ContextCompat.checkSelfPermission(activity, android.Manifest.permission.READ_EXTERNAL_STORAGE) == PackageManager.PERMISSION_GRANTED
    }

    val batteryOptimizationIgnored = pm.isIgnoringBatteryOptimizations(activity.packageName)

    val result = JSObject()
    result.put("notification", notificationGranted)
    result.put("storage", storageGranted)
    result.put("battery", batteryOptimizationIgnored)
    invoke.resolve(result)
}
```

#### 版本分支详解

| 权限 | Android 13+ (API 33+) | Android 12 及以下 |
|------|----------------------|-------------------|
| **通知** | `POST_NOTIFICATIONS` 运行时检查 | 无需运行时权限，始终 `true` |
| **存储** | `READ_MEDIA_IMAGES` 细分权限 | `READ_EXTERNAL_STORAGE` 传统权限 |
| **电池优化** | `PowerManager.isIgnoringBatteryOptimizations()` | 同上，API 无版本差异 |

> **注意**：电池优化检查**不是**标准的 Android 运行时权限模型，而是通过 `PowerManager` API 查询应用是否被列入系统电池优化白名单。

## 4. 权限请求

### 4.1 接口

```rust
#[tauri::command]
pub fn request_android_permission<R: Runtime>(app: AppHandle<R>, p_type: String) -> Result<(), String>;
```

| 属性 | 说明 |
|------|------|
| 前端命令 | `plugin:vcp-mobile\|request_android_permission` |
| 参数 | `p_type: String` —— 权限类型，支持 `"notification"` / `"storage"` / `"battery"` |
| 返回值 | `Result<(), String>` —— 成功返回 `Ok(())`，请求发起即算成功 |
| 桌面端行为 | 空操作 |

### 4.2 Kotlin 实现

```kotlin
@Command
fun requestAndroidPermission(invoke: Invoke) {
    val args = invoke.parseArgs(RequestPermissionArgs::class.java)
    when (args.type) {
        "notification" -> {
            if (Build.VERSION.SDK_INT >= 33) {
                ActivityCompat.requestPermissions(
                    activity,
                    arrayOf(android.Manifest.permission.POST_NOTIFICATIONS),
                    1001
                )
            }
        }
        "storage" -> {
            if (Build.VERSION.SDK_INT >= 33) {
                ActivityCompat.requestPermissions(
                    activity,
                    arrayOf(android.Manifest.permission.READ_MEDIA_IMAGES),
                    1002
                )
            } else {
                ActivityCompat.requestPermissions(
                    activity,
                    arrayOf(
                        android.Manifest.permission.READ_EXTERNAL_STORAGE,
                        android.Manifest.permission.WRITE_EXTERNAL_STORAGE
                    ),
                    1002
                )
            }
        }
        "battery" -> {
            try {
                val intent = Intent(Settings.ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS).apply {
                    data = Uri.parse("package:${activity.packageName}")
                }
                activity.startActivity(intent)
            } catch (e: Exception) {
                val intent = Intent(Settings.ACTION_IGNORE_BATTERY_OPTIMIZATION_SETTINGS)
                activity.startActivity(intent)
            }
        }
    }
    invoke.resolve()
}
```

#### 请求码分配

| 权限类型 | 请求码 | 说明 |
|----------|--------|------|
| `notification` | `1001` | `POST_NOTIFICATIONS` |
| `storage` | `1002` | `READ_MEDIA_IMAGES` 或 `READ_EXTERNAL_STORAGE` + `WRITE_EXTERNAL_STORAGE` |
| `battery` | 无 | 跳转系统设置页，非标准权限请求 |

#### 电池优化请求的特殊性

- **首选**：`ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` —— 直接弹出系统对话框询问用户。
- **回退**：`ACTION_IGNORE_BATTERY_OPTIMIZATION_SETTINGS` —— 跳转到系统设置页，用户需手动查找并关闭优化。

## 5. 返回桌面

### 5.1 接口

```rust

```

| 属性 | 说明 |
|------|------|
| 前端命令 | `plugin:vcp-mobile\|move_task_to_back` |
| 参数 | 无 |
| 桌面端行为 | 空操作 |

### 5.2 Kotlin 实现

```kotlin
@Command
fun moveTaskToBack(invoke: Invoke) {
    activity.moveTaskToBack(true)
    invoke.resolve()
}
```

- **`moveTaskToBack(true)`**：将当前任务移到后台，参数 `true` 表示如果当前任务是根 Activity，则整个任务进入后台；`false` 则仅当前 Activity。
- **效果**：等同于用户点击 Home 键，应用仍在运行，不触发 `onDestroy()`。

## 6. 数据流完整链路

```
前端: invoke('request_android_permission', { type: 'notification' })
    │
    ▼
system.rs :: request_android_permission("notification")
    │
    ▼
run_mobile_plugin("requestAndroidPermission", { "type": "notification" })
    │
    ▼
Kotlin: VcpMobilePlugin.requestAndroidPermission(invoke)
    │
    ├── parseArgs(RequestPermissionArgs) → { type: "notification" }
    ├── Build.VERSION.SDK_INT >= 33 ?
    │   ├── YES: ActivityCompat.requestPermissions(POST_NOTIFICATIONS, 1001)
    │   └── NO: 什么都不做
    │
    ▼
invoke.resolve()  →  前端收到 Ok(())
    │
    ▼
系统弹出权限请求对话框（Android 13+）
    │
    ▼
用户在对话框中选择 允许/拒绝
    │
    ▼
结果通过标准 Android onRequestPermissionsResult 回调
（当前实现未处理该回调，前端需通过 checkAllPermissions 轮询或下次启动时检查）
```

## 7. 关键约束与改进空间

1. **权限请求结果不可观测**：当前实现使用 `ActivityCompat.requestPermissions()`（传统 API），但**未重写 `onRequestPermissionsResult()`** 将结果回传前端。前端只能通过 `checkAllPermissions` 主动轮询或下次启动时检查权限状态。
   - **改进方向**：迁移到 Jetpack Activity Result API（`registerForActivityResult`），并在回调中通过 `evaluateJavascript` 向前端注入 `vcp-permission-result` 事件。参见 `docs/ANDROID_PLUGIN_MANAGEMENT.md` §B.2.1。

2. **`p_type` 字符串硬编码**：前端传入 `"notification"` / `"storage"` / `"battery"`，拼写错误会导致 Kotlin `when` 分支未命中，静默无操作。
   - **防范**：前端封装为枚举或常量，如 `PermissionType.NOTIFICATION`。

3. **与规范文档的交叉引用**：
   - `POST_NOTIFICATIONS` 血训：参见 `docs/ANDROID_PLUGIN_MANAGEMENT.md` §3.1 / §7.2。
   - Activity Result API 迁移模板：参见 `docs/ANDROID_PLUGIN_MANAGEMENT.md` §B.2.1。
