# 浮动窗口与会话共享

## 1. 功能概述

VCP Mobile 的浮动窗口与会话共享系统由两个核心 Kotlin 类组成，共同提供 Android 平台的"划词助手"能力：

| 类 | 文件 | 行数 | 职责 |
|---|------|:---:|------|
| `FloatingWindowManager` | `FloatingWindowManager.kt` | 259 | WindowManager 悬浮球与助手窗口的创建、显示、隐藏、拖拽、动画 |
| `ShareIntentHandler` | `ShareIntentHandler.kt` | 194 | 处理 Android 系统分享意图（ACTION_SEND / PROCESS_TEXT），提取内容并注入 WebView |

这两个类与 Rust 侧的新增命令形成完整的浮动助手链路：

```
用户在其他应用选中文本 → ShareIntentHandler 提取内容
    │
    ▼
注入 JS 事件到 AssistView WebView → floatingAssistant Store 发起对话
    │
    ▼
WebSocket → local_server.rs → agent_chat_application_service → VCP SSE 流
```

## 2. FloatingWindowManager 详解

### 2.1 架构

```
FloatingWindowManager
    ├── floatingBallView (ImageView)
    │     └── 浮动球：48dp 圆形、App 图标、半透明、可拖拽、边缘吸附动画
    │
    └── assistantContainer (LinearLayout)
          ├── handleBar (FrameLayout)
          │     └── 原生关闭把手：32dp 高、圆角顶部、半透明黑底、点击关闭
          │
          └── assistantWebView (WebView)
                ├── javascriptEnabled = true
                ├── domStorageEnabled = true
                ├── AndroidBridge JS 接口
                │     ├── closeWindow(): void
                │     └── getClipboard(): String
                └── loadUrl("http://127.0.0.1:14202/floating")
```

### 2.2 浮动球 (Floating Ball)

#### 外观参数

| 属性 | 值 | 说明 |
|------|-----|------|
| 尺寸 | 48dp × 48dp | 适配不同 DPI 屏幕 |
| 初始位置 | 屏幕右侧 x = width - 48dp - 16dp, y = height × 0.7 | 右下区域，不挡主要内容 |
| 背景 | 白色圆形 (`GradientDrawable OVAL`) | 应用图标 + 1dp 灰色描边 |
| 不透明度 | 0.85 (默认) / 1.0 (触摸中) / 0.5 (闲置) | 按触摸阶段变化 |
| 窗口类型 | `TYPE_APPLICATION_OVERLAY` (API 26+) / `TYPE_PHONE` (旧版) | WindowManager overlay |
| 窗口标志 | `FLAG_NOT_FOCUSABLE \| FLAG_LAYOUT_NO_LIMITS \| FLAG_HARDWARE_ACCELERATED` | 不抢占焦点、允许全屏、硬件加速 |

#### 触摸与拖拽

```kotlin
view.setOnTouchListener { v, event ->
    when (event.action) {
        ACTION_DOWN ->
            lastX/Y = rawX/Y; startX/Y = rawX/Y
            startTime = now(); alpha = 1.0f

        ACTION_MOVE ->
            params.x += (rawX - lastX).toInt()
            params.y += (rawY - lastY).toInt()
            边界约束: x ∈ [0, screenWidth - viewWidth], y ∈ [0, screenHeight - viewHeight]
            wm.updateViewLayout(v, params)

        ACTION_UP ->
            if (位移 < 6dp && 时间 < 300ms) → onClick()
            animateBallToEdge(v, params)
    }
    true
}
```

**拖拽设计要点**：
- `ACTION_MOVE` 中每次计算增量位移，确保跟手
- 边界约束使用 `coerceIn` 防止球体被拖出屏幕
- `ACTION_UP` 后判断是否点击（小位移 + 短时间），点击则 toggle 助手窗口，否则执行边缘吸附动画

#### 边缘吸附动画

```kotlin
private fun animateBallToEdge(view: View, params: WindowManager.LayoutParams) {
    val targetX = if (params.x + view.width / 2 < screenWidth / 2)
        0  // 左侧
    else
        screenWidth - view.width  // 右侧

    ValueAnimator.ofInt(params.x, targetX)
        .duration(300)           // 300ms 减速吸附
        .interpolator(DecelerateInterpolator)
        .addUpdateListener { ... wm.updateViewLayout(view, params) }
        .addListener { onAnimationEnd → alpha = 0.5f }
        .start()
}
```

球体总是吸附到距离最近的屏幕边缘，吸附后不透明度降至 0.5，避免干扰当前应用内容。

#### 球体可见性联动

- 助手窗口打开时：`floatingBallView.visibility = View.GONE`（球体隐藏）
- 助手窗口关闭时：`floatingBallView.visibility = View.VISIBLE`（球体重新显示）

#### AndroidBridge JS 接口

WebView 加载 `floating.html` 前，通过 `addJavascriptInterface` 注入原生桥接对象：

```kotlin
webView.addJavascriptInterface(object {
    @JavascriptInterface
    fun closeWindow() {
        activity.runOnUiThread { hideAssistantWindow() }
    }

    @JavascriptInterface
    fun getClipboard(): String {
        val cb = context.getSystemService(CLIPBOARD_SERVICE) as ClipboardManager
        return cb.primaryClip?.getItemAt(0)?.text?.toString() ?: ""

```

| 方法 | JS 调用方式 | 线程 | 用途 |
|------|-----------|------|------|
| `closeWindow()` | `AndroidBridge.closeWindow()` | 主线程 (`runOnUiThread`) | 关闭悬浮助手窗口 |
| `getClipboard()` | `AndroidBridge.getClipboard()` | WebView 线程（同步方法） | 读取系统剪贴板文本 |

#### WebView 销毁顺序

```kotlin
private fun hideAssistantWindow() {
    assistantContainer?.let { view ->
        // 1. 先从父容器移除 WebView
        assistantWebView?.let { wv ->
            (wv.parent as? ViewGroup)?.removeView(wv)
            wv.destroy()  // 2. 销毁 WebView 资源
        }
        // 3. 从 WindowManager 移除容器
        windowManager?.removeView(view)
        assistantWebView = null
        assistantContainer = null
        floatingBallView?.visibility = View.VISIBLE  // 4. 恢复悬浮球
    }
}
```

**关键设计**：先 `removeView` 再 `destroy()`，避免 Android WebView 抛出的 "destroy() called while still attached" 警告。

### 2.4 覆盖层权限管理

#### 权限检查

```kotlin
fun hasOverlayPermission(): Boolean {
    return if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
        Settings.canDrawOverlays(context)
    } else {
        true  // API < 23 无需权限
    }
}
```

Android 6.0 (API 23) 以上需要在系统设置中手动授权 `ACTION_MANAGE_OVERLAY_PERMISSION`。

#### 权限请求

```kotlin
fun requestOverlayPermission() {
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
        val intent = Intent(
            Settings.ACTION_MANAGE_OVERLAY_PERMISSION,
            Uri.parse("package:${activity.packageName}")
        ).apply { addFlags(Intent.FLAG_ACTIVITY_NEW_TASK) }
        activity.startActivity(intent)
    }
}
```

该 Intent 将用户带到系统设置页面，列出可以显示在其他应用上层的应用。应用包名通过 `package:` URI 精确定位到 VCP Mobile。

#### toggleFloatingBall 的权限守卫

```kotlin
fun toggleFloatingBall(show: Boolean): Boolean {
    if (show) {
        if (!hasOverlayPermission()) return false  // 无权限时拒绝显示
        activity.runOnUiThread { showFloatingBall() }
    } else {
        activity.runOnUiThread {
            hideAssistantWindow()
            hideFloatingBall()
        }
    }
    return true
}
```

`show == true` 时先检查权限，无权限返回 `false`（前端 `AssistantSettingsSection` 会检测此返回并重置 UI 开关状态）。

## 3. ShareIntentHandler 详解

### 3.1 支持的系统 Intent

| Intent Action | 来源场景 | 提取的内容 |
|---------------|---------|-----------|
| `ACTION_PROCESS_TEXT` | 浏览器/阅读器选中文字菜单（划词） | `intent.getCharSequenceExtra(Intent.EXTRA_PROCESS_TEXT)` |
| `ACTION_SEND` | 单文件/文本分享 | `EXTRA_TEXT`, `EXTRA_SUBJECT`, `EXTRA_STREAM` (单个) |
| `ACTION_SEND_MULTIPLE` | 多文件分享 | `EXTRA_TEXT`, `EXTRA_SUBJECT`, `EXTRA_STREAM` (多个) |

### 3.2 内容提取逻辑

```kotlin
private fun extractSharedContent(intent: Intent, context: Context): JSObject {
    val root = JSObject()

    // 1. 文本提取（优先级：PROCESS_TEXT > EXTRA_SUBJECT + EXTRA_TEXT）
    val processText = if (intent.action == Intent.ACTION_PROCESS_TEXT) {
        intent.getCharSequenceExtra(Intent.EXTRA_PROCESS_TEXT)?.toString()
    } else null

    val text = intent.getStringExtra(Intent.EXTRA_TEXT)
    val subject = intent.getStringExtra(Intent.EXTRA_SUBJECT)

    val combinedText = buildString {
        if (!processText.isNullOrBlank()) {
            append(processText)
        } else {
            if (!subject.isNullOrBlank()) { append(subject) }
            if (!text.isNullOrBlank()) {
                if (isNotEmpty() && !text.startsWith(subject ?: "")) {
                    append("\n")  // 避免 SUBJECT 与 TEXT 重复拼接
                }
                append(text)
            }
        }
    }
    root.put("text", combinedText.ifBlank { "" })

    // 2. 文件 URIs 提取
    // ACTION_SEND: intent.getParcelableExtra<Uri>(EXTRA_STREAM)
    // ACTION_SEND_MULTIPLE: intent.getParcelableArrayListExtra<Uri>(EXTRA_STREAM)
    // 每个 URI → copyStreamToCache() → { cachePath, mimeType, fileName, size }

    // 3. 文件存入缓存
    // targetPath: cacheDir/shared/shared_{timestamp}_{originalFileName}
    // 使用 contentResolver.openInputStream(uri) 读取并复制
    // bufferSize = 65536 (64KB)

    return root
}
```

### 3.3 文件缓存策略

分享的文件通过 `ContentResolver` 从 `content://` URI 读出，复制到应用内部缓存目录：

```
cacheDir/shared/
    ├── shared_1717500000_document.pdf
    ├── shared_1717500001_image.jpg
    └── ...
```

| 字段 | 来源 | 说明 |
|------|------|------|
| `cachePath` | `targetFile.absolutePath` | 缓存文件的绝对路径 |
| `mimeType` | `contentResolver.getType(uri)` | 文件的 MIME 类型 |
| `fileName` | `OpenableColumns.DISPLAY_NAME` | 原始文件名 |
| `size` | `targetFile.length()` | 文件字节数 |

### 3.4 JS 注入机制

```kotlin
fun injectShareData(webView: WebView?) {
    val data = pendingShareData ?: return
    val dataJson = data.toString()
    val safeJson = escapeJsonForJsString(dataJson)
    val script = "window.dispatchEvent(" +
        "new CustomEvent('vcp-share-intent', " +
        "{ detail: JSON.parse(\"$safeJson\") }))"
    webView.evaluateJavascript(script, null)
    pendingShareData = null  // 注入后清空缓存
}
```

| 步骤 | 说明 |
|------|------|
| `escapeJsonForJsString()` | 将 JSON 字符串中的 `\`、`"`、`'`、`\n`、`\r` 进行 JS 字符串转义 |
| `CustomEvent('vcp-share-intent')` | 标准化的事件名称，前端可通过 `addEventListener` 监听 |
| `evaluateJavascript` | WebView 方法，在当前 WebView JS 上下文中执行代码 |
| `pendingShareData` 缓存 | 若 WebView 尚未就绪，缓存数据等待后续 `onPageFinished` 或手动调用注入 |

### 3.5 延迟注入机制

```kotlin
// WebView 未就绪时缓存待注入数据
private var pendingShareData: JSObject? = null

fun handleShareIntent(intent: Intent) {
    val shareData = extractSharedContent(intent, context)
    pendingShareData = shareData
    if (webView != null) {
        injectShareData(webView)  // 立即注入
    } else {
        Log.w(TAG, "WebView not ready, caching share data")  // 缓存等待
    }
}
```

VcpMobilePlugin 在 `onPageFinished` 等 WebView 生命周期回调中调用 `shareIntentHandler.injectShareData(webView)`，确保缓存数据不会丢失。

## 8. 关键约束与血训

1. **overlay 权限不可运行时弹窗**：`SYSTEM_ALERT_WINDOW` 无法通过 `ActivityCompat.requestPermissions` 获取，必须由用户手动在系统设置中授权。前端 `AssistantSettingsSection` 在检测到无权限时自动跳转系统设置页面并重置 UI 开关。

2. **WebView 销毁必须先移除再 destroy**：Android WebView 的 `destroy()` 方法要求在 `ViewGroup` 之外调用，否则抛出"called while still attached"警告。`hideAssistantWindow()` 中严格遵循 `removeView → destroy → removeView` 顺序。

3. **浮动窗口不能使用 Tauri invoke**：WindowManager overlay 创建的 WebView 不在主 Activity 的 WebView 实例中，Tauri 的 `invoke()` JS bridge 不可用。这就是为什么浮动助手走 WebSocket + 本地服务器方案。

4. **shareIntentHandler 的延迟注入**：分享数据到达时，WebView 可能尚未创建（用户可能在主应用界面而非浮动窗口中）。`pendingShareData` 缓存确保数据不丢失，等待 WebView 就绪后注入。

5. **FileProvider 的必要性**：分享的文件通过 `ContentResolver` 复制到应用缓存目录后，若需被其他应用访问或作为附件发送，需要通过 `FileProvider` 将 file:// URI 转换为 content:// URI，否则会触发 FileUriExposedException。

6. **JSON 注入安全性**：`escapeJsonForJsString()` 对反斜杠、双引号、单引号、换行符进行转义后再拼接 JS 字符串，防止恶意构造的分享文本破坏 JS 语法或执行 XSS 攻击。
