## 3. 双轨三层架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        Vue 3 渲染层                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐  │
│  │  features/  │  │ components/ │  │  core/stores +          │  │
│  │  业务组件    │  │ 共享 UI 原语 │  │  composables            │  │
│  │  (.vue)     │  │ (.vue)      │  │  (状态 + 逻辑)           │  │
│  └──────┬──────┘  └──────┬──────┘  └───────────┬─────────────┘  │
│         │                │                      │                │
│         └────────────────┼──────────────────────┘                │
│                          │                                       │
│         ┌────────────────┴────────────────┐                      │
│         ▼                                 ▼                      │
│  ┌─────────────────┐            ┌─────────────────┐             │
│  │  Tauri IPC 桥接  │◄──────────►│  Tauri v2 API   │             │
│  │  (invoke/listen)│            │  (Channel/Event)│             │
│  └────────┬────────┘            └─────────────────┘             │
└───────────┼─────────────────────────────────────────────────────┘
            │
┌───────────▼─────────────────────────────────────────────────────┐
│                      Rust 核心层（src-tauri/）                    │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐  │
│  │ vcp_modules │  │  lib.rs     │  │  tauri-plugin-vcp-mobile│  │
│  │ 业务逻辑     │  │ 命令路由    │  │  Android 原生插件        │  │
│  └─────────────┘  └─────────────┘  └─────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

**架构原则**：

| 原则 | 说明 |
|------|------|
| **渲染层无业务逻辑** | `.vue` 文件只负责布局与事件绑定，业务逻辑委托 Store 或 Composable |
| **状态单向流动** | Rust → Store → Component，Component 通过 `invoke()` 向 Rust 发送变更请求 |
| **Backend-Driven Streaming** | 流式消息生命周期由后端 SSE 事件驱动，前端 `chatStreamStore` 仅做状态映射 |
| **Feature Co-location** | 同一领域的组件、组合式函数、类型放在同一 `features/<domain>/` 目录下 |

| 术语 | 英文/缩写 | 定义 | 相关文档 |

## 7. 与现有文档体系的关系

本 `docs/vue_docs/` 知识库与后端知识库相互独立、互补：

| 维度 | `docs/vue_docs/`（本库） | `docs/modules/`（Rust 模块库） | `docs/sync/`（同步库） |
|------|------------------------|------------------------------|----------------------|
| 覆盖范围 | Vue 3 前端全部源码（22 份） | 16 个稳定 Rust 模块 | 同步 V2 子系统全链路 |
| 视角 | UI 状态、组件交互、Pinia 数据流 | 业务逻辑、数据库、网络协议 | WebSocket + HTTP + Hash 差异 |
| 阅读对象 | 前端开发者、UI 调试者 | 后端开发者、新成员 | 同步功能开发者 |
| 典型内容 | `chatStreamStore` 设计、`ChatView.vue` 布局 | `vcp_client.rs`、`db_manager.rs` | `sync_service.rs`、`sync_pipeline` |

**交叉引用示例**：
- 前端文档提到后端模块时：`详见 [VCP 请求客户端](../modules/09_VCP请求客户端.md)`
- 前端文档提到同步协议时：`详见 [同步架构总览](../sync/01_架构总览与设计理念.md)`
