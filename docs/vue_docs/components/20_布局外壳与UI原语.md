# 20. 布局外壳与UI原语

## 1. 概述

### 1.1 领域定位

`src/components/layout/` 与 `src/components/ui/` 构成 VCP Mobile 前端**空间结构层**与**交互原语层**。`layout/` 负责应用级的空间划分（左右抽屉、启动屏、权限门禁），决定用户在宏观尺度上的导航与视野边界；`ui/` 提供可复用的原子级交互容器（弹层、页面、通知、编辑器等），是各 Feature 模块构建具体业务界面的基础材料。

该层级的边界严格限定为：
- **空间结构**：侧边栏抽屉、启动加载层、权限引导页
- **交互原语**：BottomSheet、SlidePage、Toast、Prompt、全屏编辑器等通用容器
- **全局调度**：覆盖层的集中挂载与层级管理

该层级**不**涉及：
- 具体业务表单（由 `features/settings/` 等承担）
- 消息渲染逻辑（由 `features/chat/` 承担）
- 数据持久化（由 Rust 后端承担）

### 1.2 模块构成表

| 目录 | 文件数 | 总代码量 | 职责 |
|------|--------|---------|------|
| `src/components/layout/` | 4 | ~684 行 | 应用级空间结构：左侧抽屉、右侧抽屉、启动屏、权限门禁 |
| `src/components/ui/` | 13 | ~1,475 行 | 交互原语：BottomSheet、SlidePage、Toast、Prompt、编辑器等 |
| `src/components/` 顶层 | 3 | ~529 行 | 全局覆盖层调度器、业务挂载点、模型选择器 |

### 1.4 核心设计原则

| 原则 | 说明 |
|------|------|
| **语义化层级** | 所有覆盖层组件通过 `src/core/constants/layers.ts` 获取 z-index，禁止在任何组件中直接使用 `z-50`、`z-[999]` 等魔法数字 |
| **物理返回键拦截 (Operation Aegis)** | 所有模态组件通过 `useModalHistory` 注册到浏览器 history stack，确保 Android 物理返回键/侧滑手势优先关闭弹窗，而非直接退出应用 |
| **手势优先** | 侧边栏滑动关闭、BottomSheet 下滑关闭、Toast 左右滑动关闭，均基于 `@vueuse/core` 的 `useSwipe` 实现 |
| **常驻 DOM 策略** | SettingsView、AgentSettingsView 等通过 `FeatureOverlays` 保持常驻 DOM（`v-show` 控制），确保 leave 动画完整、表单草稿不丢失 |
| **安全区适配** | 所有固定定位的底部/顶部元素均通过 `env(safe-area-inset-*)` 适配刘海屏与手势条，禁止硬编码 `padding-top: 20px` |
| **Teleport 脱离文档流** | 模态组件统一使用 `<Teleport to="body">`（或 `#vcp-feature-overlays`），避免受父级 `overflow: hidden` 或 `transform` 裁剪影响 |

## 2. 布局外壳（layout/）

### 2.1 AgentSidebar — 左侧抽屉

> 文件位置：`src/components/layout/AgentSidebar.vue`（157 行）

AgentSidebar 是应用的主导航抽屉，承载智能体与话题的浏览、搜索、新建入口。

#### 三栏布局

```
┌─────────────────────────────┐
│  VCP MOBILE                 │  ← 头部：品牌标题 + SidebarTabs + SidebarSearch
│  [Agents] [Topics]          │
│  ┌─────────────────────┐    │
│  │ Search...           │    │
│  └─────────────────────┘    │
├─────────────────────────────┤
│                             │
│  ● Nova                    │  ← 列表区：AgentList / TopicList
│  ● 艾米莉亚                 │
│  ● 测试助手                 │
│                             │
├─────────────────────────────┤
│  [+ 新建助手]               │  ← 底部：动作区 + 全局设置按钮
│  [ 全局设置  > ]            │
└─────────────────────────────┘
```

- **头部**：`SidebarTabs`（`agents`/`topics` 切换）+ `SidebarSearch`（实时过滤）
- **列表区**：`activeTab === 'agents'` 时渲染 `AgentList`；`activeTab === 'topics'` 时渲染 `TopicList`
- **底部**：根据当前 Tab 显示 `AgentsCreator` 或 `TopicCreator`，外加全局设置快捷入口

#### 状态与手势

与 `layoutStore.leftDrawerOpen` 单向绑定：

```vue
<aside :class="{ 'is-open': layoutStore.leftDrawerOpen }">
```

内部通过 `useSidebarSwipe` 监听左滑关闭、右滑切回 agents Tab：

```ts
useSidebarSwipe(sidebarRef, {
  type: 'left',
  onTabSwitch: () => { activeTab.value = 'agents'; }
});
```

#### CSS 硬件加速动画

```css
.vcp-drawer {
  transition: transform 0.4s cubic-bezier(0.16, 1, 0.3, 1);
  transform: translateX(-100%);        /* 关闭态 */
  z-index: var(--layer-drawer);        /* 20 */
}
.vcp-drawer-left.is-open {
  transform: translateX(0);            /* 打开态 */
}
```

使用 `transform` 而非 `left` 属性，触发 GPU 合成层，在低端 Android 设备上仍保持 60fps。

#### 桌面端常驻适配

```css
@media (min-width: 768px) {
  .vcp-drawer {
    position: relative;
    transform: translateX(0) !important;
    width: 280px;
    z-index: var(--layer-local);        /* 降级为 10，不参与覆盖层竞争 */
  }
}
```

在桌面端，左侧边栏从"覆盖层"降级为"布局的一部分"，不再占用 drawer 层级，避免与右侧内容产生 z-index 冲突。

### 2.3 BootScreen — 启动屏

> 文件位置：`src/components/layout/BootScreen.vue`（67 行）

BootScreen 是应用冷启动时的全屏加载层，与 `appLifecycleStore.state` 深度绑定。

#### 两态设计

| 状态条件 | 视觉表现 |
|----------|----------|
| `state !== 'READY' && state !== 'ERROR'` | 蓝色旋转 Spinner + `statusText` + `currentPhaseLabel` + 当前 state 标签 |
| `state === 'ERROR'` | 红色错误卡片 + 错误详情 + "重试启动"按钮 |

```vue
<!-- 加载态 -->
<div v-if="lifecycleStore.state !== 'READY' && lifecycleStore.state !== 'ERROR'"
     class="fixed inset-0 z-boot ...">
  <div class="animate-spin">...</div>
  <h2>{{ lifecycleStore.statusText }}</h2>
  <p>{{ lifecycleStore.currentPhaseLabel }}</p>
</div>

<!-- 错误态 -->
<div v-if="lifecycleStore.state === 'ERROR'"
     class="fixed inset-0 z-boot ...">
  <div class="border-red-500/20 ...">
    <button @click="reloadApp()">重试启动</button>
  </div>
</div>
```

#### 状态流转关联

BootScreen 本身不驱动状态机，仅被动响应 `appLifecycleStore`：

```
PERMISSIONS → BOOTING → CONNECTING → PRELOADING → READY
                ↓
              ERROR
```

当状态到达 `READY` 时，加载层通过 Vue `<Transition name="fade">` 淡出（0.3s）；若进入 `ERROR`，则切换为错误看板。

### 2.4 PermissionGate — 权限门禁

> 文件位置：`src/components/layout/PermissionGate.vue`（255 行）

PermissionGate 是 Android 首次启动时的**全屏权限引导门禁**。只有在 `notification` + `storage` + `battery` 三项权限全部获取后，用户才能进入应用主界面。

#### 三步骤向导

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1          Step 2          Step 3                     │
│   ●──────────────●──────────────○                           │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  授予权限                                            │   │
│  │  ┌────────┐ ┌────────┐ ┌────────┐                  │   │
│  │  │通知 去授权│ 存储 去授权│ 电池 去授权│                  │   │
│  │  └────────┘ └────────┘ └────────┘                  │   │
│  │            [ 下一步 ]                               │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

- **Step 1（权限授予）**：通知、存储空间、后台运行三项权限卡片，每项显示状态（OK / 去授权按钮）
- **Step 2（电池白名单）**：解释品牌厂商电池管理策略，引导用户进入设置页添加白名单
- **Step 3（占位）**："更多功能即将上线"

#### 与 Rust 后端的 IPC

```ts
// 批量检查权限
const res = await invoke<PermissionStatus>('plugin:vcp-mobile|check_all_permissions');

// 请求单项权限
await invoke('plugin:vcp-mobile|request_android_permission', { pType: type });

// 退出到后台（用户拒绝权限时）
await invoke('plugin:vcp-mobile|move_task_to_back');
```

#### 三重检测机制

1. **主动检查**：`onMounted` 立即调用 `check()`
2. **事件驱动**：监听 Kotlin 侧通过 `evaluateJavascript` 注入的 `vcp-permission-change` 自定义事件
3. **轮询兜底**：10 秒间隔 `setInterval(check, 10000)`，防止极端情况下事件丢失

```ts
onMounted(() => {
  check();
  window.addEventListener('visibilitychange', onVisibilityChange);
  window.addEventListener('vcp-permission-change', onPermissionChange);
  checkTimer = setInterval(check, 10000);
});
```

层级：`z-gate`（110），全局最高，确保权限未授予前用户无法通过任何手段绕过。

### 3.1 BottomSheet — 底部弹层

> 文件位置：`src/components/ui/BottomSheet.vue`（115 行）

BottomSheet 是从底部滑出的模态操作面板，用于呈现一组相关的操作选项。

#### 结构与层级

```vue
<Teleport to="body">
  <!-- 遮罩层 -->
  <div v-if="modelValue" class="fixed inset-0 bg-black/50 z-sheet" @click="close">
  </div>
  <!-- 内容区 -->
  <div v-if="modelValue" class="fixed bottom-0 left-0 right-0 z-sheet ...">

    <button v-for="action in actions">...</button>
    <button @click="close">取消</button>
  </div>
</Teleport>
```

- **层级**：`z-sheet`（50）
- **遮罩**：`bg-black/50`，点击关闭
- **安全区**：`padding-bottom: calc(env(safe-area-inset-bottom, 20px) + 16px)` 适配 iPhone 刘海/灵动岛

#### 操作项接口

```ts
export interface ActionItem {
  label: string;
  icon?: any;            // lucide-vue-next component
  danger?: boolean;      // 红色警示样式
  disabled?: boolean;
  handler: () => void;
}
```

#### 模态历史集成

通过 `useModalHistory` 注册，支持物理返回键关闭：

```ts
watch(() => props.modelValue, (newVal) => {
  if (newVal) {
    registerModal('BottomSheet', () => emit('update:modelValue', false));
  } else {
    unregisterModal('BottomSheet');
  }
});
```

### 3.2 SlidePage — 虚拟页面

> 文件位置：`src/components/ui/SlidePage.vue`（39 行）

SlidePage 是**虚拟页面栈**的原子组件。VCP Mobile 本质上是单页应用（仅 `/chat` 一条路由），所有"页面跳转"（如设置页、Agent 设置页）均通过 SlidePage 从右侧滑入模拟。

#### 核心实现

```ts
const props = withDefaults(defineProps<Props>(), {
  zIndex: LAYER_PAGE_BASE,    // 40
});
```

```vue
<Transition name="slide-page">
  <div v-show="props.isOpen" class="fixed inset-0 pointer-events-auto" :style="{ zIndex: props.zIndex }">
    <slot />
  </div>
</Transition>
```

#### 动态层级

z-index 由 `overlayStore.getPageZIndex(type)` 动态计算：

```ts
const getPageZIndex = (type: string) => {
  const index = pageStack.value.findIndex(p => p.type === type);
  if (index === -1) return LAYER_PAGE_BASE;
  return LAYER_PAGE_BASE + Math.min(index, LAYER_PAGE_MAX_OFFSET);  // 封顶 49
};
```

这确保后打开的页面（如从设置页进入 Agent 设置）具有更高的 z-index，视觉上覆盖前者。

#### 动画曲线

```css
.slide-page-enter-active {
  transition: transform 0.35s cubic-bezier(0.32, 0.72, 0, 1);
}
.slide-page-leave-active {
  transition: transform 0.3s cubic-bezier(0.32, 0.72, 0, 1);
}
```

采用 iOS 风格的 `cubic-bezier(0.32, 0.72, 0, 1)`（俗称"流畅曲线"），起始段快速响应，末尾段柔和减速，在 120Hz 屏幕上呈现原生级的滑动手感。

#### 物理返回键集成（Operation Aegis）

`FeatureOverlays` 中每个 SlidePage 的关闭回调都会调用 `overlayStore.popPage()`，后者通过 `unregisterModal` 与浏览器 history stack 同步，实现物理返回键逐层关闭页面。

### 3.3 ToastManager / ToastItem — 通知气泡

ToastManager 文件：`src/components/ui/ToastManager.vue`（38 行）  
ToastItem 文件：`src/components/ui/ToastItem.vue`（90 行）

#### ToastManager

```vue
<div class="fixed top-safe left-0 right-0 z-toast pointer-events-none px-4 pt-4 flex flex-col items-center gap-2.5">
  <TransitionGroup name="toast">
    <ToastItem v-for="toast in store.activeToasts" :key="toast.id" :toast="toast" />
  </TransitionGroup>
</div>
```

- **层级**：`z-toast`（90）
- **容器**：`pointer-events-none`，仅子项恢复 `pointer-events-auto`
- **列表动画**：`TransitionGroup`，enter 时从上方弹入带弹性（cubic-bezier(0.18, 0.89, 0.32, 1.28)），leave 时向上淡出

#### ToastItem

**类型与视觉映射**：

| 类型 | 图标 | 颜色 |
|------|------|------|
| `success` | `CheckCircle` | `text-green-500` |
| `warning` | `AlertTriangle` | `text-amber-500` |
| `error` | `X` | `text-red-500` |
| `tool` | `Cpu` | `text-purple-500` |
| `agent` | `User` | `text-blue-500` |
| 默认 | `Info` | `text-blue-400` |

**交互**：
- 点击整项关闭
- 左右滑动关闭（`useSwipe`，`Math.abs(lengthX) > 60`）
- 手动关闭按钮（右上角 X）
- 滑动时实时计算 opacity：`opacity = Math.max(0, 1 - Math.abs(lengthX) / 200)`

**自动消失**：由 `notificationStore.addNotification` 管理，默认 3000ms，`duration: 0` 表示永不消失（用于审批类通知）。

#### 单例抑制与冷却

`notificationStore` 实现了 P0 级优化：
1. 同 ID 的 Toast 已在展示中 → 原地更新，不新建
2. 同 ID 在 30 秒内已在历史中出现 → 抑制新 Toast，仅更新时间戳

### 3.4 VcpPrompt / ContextMenuSheet — 对话框与上下文菜单

单文本输入确认对话框，用于重命名、快速输入等场景。

<Teleport to="body">

- **键盘支持**：Enter 确认、Esc 取消
- **模态动画**：外层 fade + 内层 scale + translateY 弹性效果

#### ContextMenuSheet

底部弹出的上下文操作菜单，通常由长按触发。

```vue
<Teleport to="body">
  <div v-if="isOpen" class="fixed inset-0 bg-black/20 pointer-events-auto z-dialog">
    <div class="absolute left-1/2 bottom-6 -translate-x-1/2 ...">
      <button v-for="action in actions" @click="handleAction(action)">
        <component :is="action.icon" />
        <span>{{ action.label }}</span>
      </button>
    </div>
  </div>

```

与 BottomSheet 的区别：
- 视觉更紧凑，圆角更小（`rounded-3xl` vs `rounded-t-[2rem]`）
- 操作项为横向图标 + 文本，非大按钮
- 无独立"取消"按钮，点击遮罩关闭

### 3.6 FullScreenEditor — 全屏编辑器

> 文件位置：`src/components/ui/FullScreenEditor.vue`（147 行）

用于编辑长文本（如 Agent 系统提示词）的全屏文本编辑器。

#### 关键特性

**1. 软键盘适配**

```ts
const { keyboardHeight } = useKeyboardInsets();
// 动态 paddingBottom
:style="{ paddingBottom: `calc(env(safe-area-inset-bottom) + ${keyboardHeight}px)` }"
```

软键盘弹出时，通过 `watch(keyboardHeight)` 重新定位光标，确保输入区域始终可见。

**2. Rubber-band 防护**

Android/iOS WebView 中，textarea 滚动到边界后继续滑动会导致整个页面被拖动（rubber-band 效应）。`FullScreenEditor` 通过 `touchmove` 拦截实现三态防护：

```ts
// 内容未溢出 → 直接阻止
if (scrollHeight <= clientHeight) { e.preventDefault(); return; }

// 在顶部且继续下滑 → 阻止
if (scrollTop <= 0 && deltaY < 0) { e.preventDefault(); return; }

// 在底部且继续上滑 → 阻止
if (scrollTop + clientHeight >= scrollHeight - 1 && deltaY > 0) { e.preventDefault(); return; }
```

**3. Android WebView 优化**

```css
.vcp-fullscreen-textarea {
  touch-action: manipulation;
  -webkit-tap-highlight-color: transparent;
  cursor: text;
  overscroll-behavior: contain;
}
```

层级：`z-viewer`（70），高于 dialog 层，确保编辑器打开时覆盖所有业务弹窗。

### 3.8 VcpAvatar — 头像组件

> 文件位置：`src/components/ui/VcpAvatar.vue`（111 行）

头像展示组件，支持图片、首字母占位、Dominant Color 背景三种渲染策略。

#### 三级渲染策略

```
┌─────────────────────────────┐
│  ┌───────────────────────┐  │
│  │  Fallback 层 (底层)    │  │  ← 首字母 + 确定性背景色
│  │        "N"             │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │  图片层 (顶层，条件渲染)│  │  ← avatarStore.getAvatarUrl()
│  │   [实际头像图片]        │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
```

**1. 图片层**：
- `avatarStore.getAvatarUrl(ownerType, ownerId, version)` 异步获取 Blob URL
- **缓存优先**：同步检查 `avatarStore.cache`，命中则立即显示，消除加载"顿一下"的感觉
- `imgExists` 控制渲染；`@error="handleImgError"` 触发降级

**2. Fallback 层**：
- `initial`：取 `fallbackName` 或 `ownerId` 的首字母大写
- `fallbackBg`：基于 `ownerId` 字符串 hash 从 5 色轮盘中选取确定性颜色（红/蓝/绿/琥珀/紫）
- 若传入 `dominantColor`，则直接使用主色调作为背景

**3. 主色调边框**：

```ts
const borderStyle = computed(() => {
  if (!color) return {};
  const mixedColor = `color-mix(in srgb, ${color} 80%, transparent)`;
  return {
    borderColor: mixedColor,
    boxShadow: `0 0 8px ${color}33`
  };
});
```

使用 CSS `color-mix` 混合出 80% 透明度边框 + 减弱发光，使头像边缘自然融入背景。

### 3.11 WebGLFluidBackground — WebGL 流体背景

> 文件位置：`src/components/ui/WebGLFluidBackground.vue`（351 行）

WebGL 实时渲染的流体极光背景，用于启动屏、空态页或特定主题页面。

#### 渲染管线

```
Vertex Shader (全屏 Quad)
         │
         ▼
Fragment Shader
  ├─ Domain Warping (UV 正弦扰动，产生液态凝胶边界)
  ├─ 4 条异向高斯极光带 (Cyan/Magenta/Violet/Blue)
  ├─ Isotropic Repulsion (鼠标/触摸排斥力场)
  ├─ 气态光学逐层叠染 (正向 mixing，消除互补色脏污)
  └─ 暗色模式自适应 (u_is_dark uniform)
```

#### 关键参数

| 参数 | 数值 | 说明 |
|------|------|------|
| 排斥力半径 | 25% 屏幕宽度 | 触摸点周围产生流体扰动的范围 |
| 最大位移 | 15% 屏幕宽度 | 极光带被排斥的最大距离 |
| DPR 限制 | 2 | 避免高 DPR 设备 GPU 过载 |
| 颜色强度（亮色模式） | 0.35–0.40 | 确保亮色背景下不刺眼 |
| 颜色强度（暗色模式） | 0.45–0.52 | 暗色背景下增强表现力 |

#### 性能与生命周期

- **启动延迟**：`setTimeout(initWebGL, 100)`，让浏览器先完成布局与样式计算
- **尺寸同步**：`ResizeObserver` 监听父元素，完美解决 SlidePage 缓动滑入期间宽度动态变更导致的 Canvas 尺寸锁定问题
- **卸载清理**：`onUnmounted` 中调用 `WEBGL_lose_context` 立即释放 GPU 内存，取消 `requestAnimationFrame`
- **主题切换**：`watch(themeStore.isDarkResolved)` 直接更新 uniform，无需重建 shader

#### 低电量模式降级建议

WebGLFluidBackground 持续占用 GPU。在电池电量低于 20% 或系统开启省电模式时，建议父组件通过条件渲染将其替换为静态 CSS 渐变背景。

## 4. 全局覆盖层管理

### 4.1 GlobalOverlayManager — 覆盖层调度器

> 文件位置：`src/components/GlobalOverlayManager.vue`（52 行）

GlobalOverlayManager 是全局覆盖层的集中调度与挂载点，职责是协调 Prompt、ContextMenu、Editor、Toast 等基础 UI 的显隐状态。

```vue
<div class="fixed inset-0 pointer-events-none z-toast">
  <!-- 基础 UI -->
  <VcpPrompt v-if="overlayStore.promptConfig" class="pointer-events-auto" ... />
  <ContextMenuSheet v-if="overlayStore.contextMenuConfig" ... />
  <FullScreenEditor v-if="overlayStore.editorConfig" class="pointer-events-auto" ... />
  <ToastManager class="pointer-events-auto" />

  <!-- Feature 投射目标 -->
  <div id="vcp-feature-overlays" class="absolute inset-0 pointer-events-none"></div>
</div>
```

#### 设计要点

1. **容器 `pointer-events-none`**：自身不拦截任何交互，由子组件通过 `pointer-events-auto` 恢复
2. **ToastManager 不离散 Teleport**：直接渲染在容器内，确保始终位于 `z-toast` 层级
3. **其他组件内部 Teleport**：VcpPrompt、ContextMenuSheet、FullScreenEditor 内部均使用 `<Teleport to="body">`，实际渲染位置脱离 GlobalOverlayManager，不受其 z-index 约束
4. **`#vcp-feature-overlays`**：供 `AvatarCropper` 等业务组件通过 Teleport 投射，避免与 body 内其他元素产生层级竞争

### 4.2 FeatureOverlays — 业务覆盖层挂载点

> 文件位置：`src/components/FeatureOverlays.vue`（77 行）

FeatureOverlays 是所有全局业务视图的统一挂载点，核心设计目标是**平衡 DOM 常驻与按需渲染**。

#### 常驻 DOM 策略

以下组件保持常驻（`v-show` 由 `is-open` 控制），以保留组件本地状态：

| 组件 | 状态控制 | 保留的状态 |

| `SettingsView` | `overlayStore.isSettingsOpen` | 表单草稿、滚动位置 |

### 5.2 层级冲突规避策略

1. **SlidePage 动态递增**  
   `LAYER_PAGE_BASE + stackIndex`，最多叠加 9 层（封顶 49）。后打开的页面（如从设置页进入 Agent 设置）始终覆盖前者，避免页面栈中的层级倒置。

2. **抽屉互斥**  
   `layoutStore.setLeftDrawer(true)` 会自动调用 `setRightDrawer(false)`，反之亦然。两侧抽屉不会同时打开，避免 82vw 宽度在窄屏上产生视觉挤压和交互冲突。

3. **Toast 独立于模态历史栈**  
   Toast 不注册到 `modalHistory`，不拦截物理返回键。这确保用户快速关闭弹窗（如 BottomSheet）时，不会被 Toast 阻碍；同时 Toast 的滑动关闭属于独立交互通道。

4. **Teleport 脱离文档流**  
   BottomSheet、VcpPrompt、ContextMenuSheet、UpdatePrompt 均 `Teleport to="body"`，其层级不受父级 `overflow: hidden`、`transform` 或 `opacity` 的影响，从根本上消除裁剪问题。

5. **FeatureOverlays 内部无 DOM 顺序竞争**  
   所有 SlidePage 共享同一个 `fixed inset-0` 容器，靠动态 `zIndex` 区分前后关系，不依赖 Vue 渲染顺序或 DOM 插入顺序。

## 7. 设计决策与注意事项

### 7.1 为何使用 `useModalHistory` 而非 Android 原生返回拦截？

Operation Aegis 选择基于浏览器 `history.pushState` 的栈管理，而非 Android `onBackPressed`：
1. **跨平台一致性**：同一套逻辑在桌面浏览器调试和 Android 真机行为完全一致
2. **与 Vue Router 共存**：仅在主界面（`/` 或 `/chat`）拦截，真实路由页返回交给 vue-router
3. **状态机防重入**：`IDLE` / `POPSTATE_HANDLING` / `INTERNAL_BACK` 三态机避免 `history.back()` 递归调用

### 7.2 常驻 DOM vs 按需渲染的权衡

`FeatureOverlays` 中采用混合策略：
- **SettingsView 等常驻**：保留表单草稿、滚动位置、组件内部缓存；确保 `SlidePage` 的 leave 动画（0.3s 滑出）有完整的 DOM 生命周期
- **SyncSessionView `v-if`**：同步会话状态完全由 `syncSessionStore` 托管，会话结束后组件实例应完全销毁以释放 WebSocket 连接和内存；且同步界面不含需要保留的表单草稿

### 7.3 BottomSheet 与 ContextMenuSheet 的区分

| 维度 | BottomSheet | ContextMenuSheet |
|------|-------------|------------------|
| 视觉 | 大圆角（`rounded-t-[2rem]`）+ 拉手条 | 小圆角（`rounded-3xl`）+ 更紧凑 |
| 操作项 | 带图标的大按钮，占据整行，支持 `danger` | 图标 + 文本的横向列表 |
| 取消按钮 | 内置独立"取消"大按钮 | 无，点击遮罩关闭 |
| 典型场景 | 模型选择、主要操作分支 | 长按菜单、次要操作 |

### 7.4 AvatarCropper 的 Ref 探测防御

`vue-cropper` 在不同版本下 Ref 结构不同（直接 ref vs `.value` 嵌套）。`AvatarCropper` 采用运行时探测而非编译时假设：

```ts
const realCropper = cropper.value.getCropBlob
  ? cropper.value
  : cropper.value.value;
```

这种防御性编程避免了依赖升级导致的裁剪功能完全失效，是移动端 WebView 环境中"不信任依赖 ABI"原则的体现。

### 7.5 WebGLFluidBackground 的触摸监听策略

事件监听器绑定在 **Canvas 父元素**而非 Canvas 本身：

```ts
boundParent.addEventListener('touchmove', trackInteraction, { passive: false });
```

原因：Canvas 自身设置 `pointer-events-none`，且父元素通常是被覆盖的完整视口容器。通过父元素监听实现了"零侵入"——组件不抢占交互，只读取坐标驱动视觉效果。

### 7.6 AgentSidebar 桌面端层级降级

桌面端 `md` 断点下，AgentSidebar 从 `z-drawer`（20）降级为 `z-local`（10），且 `position: relative`。这一设计决策确保：
1. 侧边栏不再产生"覆盖"右侧内容的行为，而是与内容区并列布局
2. 不再参与移动端的手势关闭逻辑（`transition: none`）
3. 降低层级避免与可能出现的 dialog/sheet 产生不必要的竞争

### 7.7 ModelSelector 的定位说明

`ModelSelector.vue`（位于 `src/components/ModelSelector.vue`）在视觉与交互上属于 BottomSheet 的特化实例，已在文档 `23_ModelSelector与模型管理.md` 中详细覆盖。其核心技术特征包括：VueUse `useVirtualList` 虚拟滚动（固定 62px itemHeight）、厂商 Tag 动态聚合、120px 动力学下滑关闭阈值、以及骨架屏 Shimmer 动画。本节仅将其纳入全局层级表（`z-sheet`，50），不再展开细节。
