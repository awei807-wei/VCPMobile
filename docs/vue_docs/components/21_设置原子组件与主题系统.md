# 21. 设置原子组件与主题系统

## 1. 概述

### 1.1 领域定位

`components/settings/` 与 `features/settings/ThemePicker.vue` 构成 VCP Mobile 前端**设置领域的视觉层底座**。该领域不负责业务逻辑（业务逻辑由各自领域的 Store 与 Tauri Command 承担），仅提供：

1. **原子 UI 组件**：表单输入、开关、选择器、按钮、状态指示、布局容器等设置页专用原语；
2. **主题运行时系统**：基于 CSS 变量的动态主题切换、明暗模式解析、壁纸层管理；
3. **UnoCSS 配置层**：原子类引擎的预设、快捷类、断点与层级映射。

### 1.2 模块构成表

| 文件/目录 | 类型 | 职责 |
|-----------|------|------|
| `src/components/settings/*.vue` (9 个) | 原子组件 | 设置页专用 UI 原语 |
| `src/assets/themes.css` | 全局 CSS | `:root` 回退变量、玻璃态面板、消息气泡基础样式 |
| `src/assets/themes/*.ts` (13 个) | 主题模块 | 各主题的暗色/亮色变量与额外 CSS |
| `src/uno.config.ts` | 配置文件 | UnoCSS 引擎预设、快捷类、主题色、断点、层级 |
| `src/core/stores/theme.ts` | Pinia Store | 主题加载、切换、HMR、壁纸缩略图缓存 |
| `src/core/constants/layers.ts` | TS 常量 | 语义化 z-index 数值常量 |
| `src/features/settings/ThemePicker.vue` | 业务组件 | 主题网格/横向选择器、壁纸预览 |

### 1.4 核心设计原则

| 原则 | 说明 |
|------|------|
| **原子化拼装** | 设置页不依赖巨型"设置表单"组件，而是由 `SettingsRow` + `SettingsTextField` / `SettingsSwitch` 等原子横向拼装，保持布局一致性 |
| **CSS 变量驱动主题** | 全部色彩、背景、边框通过 `--*` CSS 变量运行时注入，切换主题无刷新、无 DOM 重建 |
| **TypeScript 主题模块** | 主题从传统的 `.css` 文件迁移为 `.ts` 模块，支持 Vite HMR、类型检查、动态导入与 Tree Shaking |
| **双重层级定义** | `layers.ts` (TS 常量) ↔ `uno.config.ts` (zIndex 预设) ↔ `themes.css` (CSS 变量) 三重映射，消除魔法数字 |
| **快捷类克制** | UnoCSS shortcuts 仅保留 4 个通用组合（`btn`, `card`, `flex-center`, `flex-col-center`），禁止定义领域专用 shortcuts，防止命名冲突与隐式依赖 |

## 2. 设置原子组件体系

### 2.1 组件清单与职责表

`src/components/settings/` 目录下共 9 个 `.vue` 单文件组件，全部使用 `<script setup>` 语法。

| 组件 | 职责 | 核心 Props | 使用场景 |
|------|------|-----------|----------|
| `SettingsTextField` | 带标签的文本输入框，支持安全遮罩、错误态、居中 | `modelValue`, `label`, `placeholder`, `mono`, `isSecure`, `error`, `center` | API Key 输入、模型参数配置、数值输入 |
| `SettingsSwitch` | iOS 风格滑动开关 | `modelValue`, `disabled`, `activeColor` | 功能开关（流式输出、同步开关等） |
| `SettingsDisclosure` | 可展开/折叠的面板，带标题、描述、图标 | `title`, `description`, `icon`, `defaultOpen`, `accentColor` | 高级选项折叠、分组子设置 |

### 2.2 表单输入封装

**`SettingsTextField.vue`**（106 行）是设置页最复杂的原子组件，封装了移动端表单输入的全部细节：

```
┌─────────────────────────────────────────────┐
│  label (可选, 10px uppercase, opacity-40)    │
│  ┌───────────────────────────────────────┐  │
│  │  input 区域                            │  │
│  │  [██████░░░░░░░░░░░░░░░░░]  [👁/⚠️]   │  │
```

> 文件位置：`src/components/settings/SettingsTextField.vue`

| 特性 | 实现方式 |
|------|----------|
| 安全遮罩 | `isSecure` 模式下使用 `-webkit-text-security: disc` 与 focus 状态联动，点击眼睛图标切换可见性 |
| 错误态 | `error=true` 时边框变红 + 右侧显示警告图标 |
| Monospace | `mono=true` 或 `isSecure=true` 时应用 `font-mono text-sm`，适合 API Key、Token 等 |
| 自动禁用智能输入 | `autocomplete="off" autocapitalize="off" spellcheck="false"` |

### 2.3 开关与选择器

**`SettingsSwitch.vue`**（31 行）实现纯 CSS 的 iOS 风格开关：

```
关闭态:  ●━━━━○          开启态:  ━━━━○●
        bg-black/10              bg-blue-500 (或 activeColor)
```

- 无外部依赖，仅使用 Tailwind/UnoCSS 原子类；
- 通过 `translate-x-5` / `translate-x-0` 实现滑块动画；
- `active:scale-90` 提供触觉反馈。

**`SettingsDisclosure.vue`**（60 行）是设置页的"手风琴"选择器：

- 使用 Vue `<Transition>` + `max-h-0` / `max-h-[2000px]` 实现无 JS 动画的展开折叠；
- 标题行右侧 Chevron 图标通过 `rotate-180` 旋转指示状态；
- 展开时底部出现 `border-t` 分隔线，内容区通过 slot 注入任意子组件。

## 3. 主题 CSS 变量体系

### 3.1 themes.css 结构

`src/assets/themes.css`（131 行）承担**全局回退层**角色，包含三类内容：

1. **`:root` 回退变量**：在主题 TS 模块加载前提供基础色彩，防止白屏闪烁；
2. **布局与交互基础样式**：`html/body/#app` 尺寸、`overscroll-behavior`、安全区；
3. **组件级基础样式**：抽屉阴影、图标强控、自定义滚动条、玻璃态面板、消息气泡。

> 文件位置：`src/assets/themes.css`

### 3.2 明暗模式变量映射

项目**不**使用传统的 `.dark {}` 静态覆盖，而是采用**运行时注入**策略：

```
┌─────────────────┐
│   主题 TS 模块   │
│  ┌───────────┐  │
│  │ variables │  │
│  │  dark: {} │  │───► themeStore 根据 isDarkResolved
│  │ light: {} │  │     选择 dark/light 分支
│  └───────────┘  │
└────────┬────────┘
         │ injectVariables()
         ▼
┌─────────────────────┐
│ document.documentElement.style.setProperty('--key', 'value')
│
│ --primary-bg: #1c1c1e
│ --secondary-bg: #28282c
│ --primary-text: #e0e0e0
│ ... (约 30+ 个变量)
└─────────────────────┘
```

每个主题 TS 模块（如 `themes-simple-bw.ts`）导出：

```ts
export const meta = { fileName: 'themes-simple-bw.css', name: '黑白简约' };
export const variables = { dark: { '...': '...' }, light: { '...': '...' } };
export const extraCss = `.tool-bubble { ... }`;
```

典型变量清单（以 `themes-simple-bw.ts` 为例）：

| 变量名 | 暗色值 | 亮色值 | 用途 |
|--------|--------|--------|------|
| `--primary-bg` | `#1c1c1e` | `#f4f6f8` | 页面主背景 |
| `--secondary-bg` | `#28282c` | `#ffffff` | 卡片/面板背景 |
| `--tertiary-bg` | `#121212` | `#e9edf0` | 最深/最浅层级背景 |
| `--accent-bg` | `#3a3a3e` | `#e0e6eb` | 强调背景、悬停态 |
| `--border-color` | `#3a3a3e` | `#e0e6eb` | 全局边框 |
| `--primary-text` | `#e0e0e0` | `#2c3e50` | 主文字色 |
| `--secondary-text` | `#a0a0a0` | `#5a6f80` | 次要文字 |
| `--highlight-text` | `#6fa8dc` | `#3498db` | 链接/高亮 |

### 3.3 壁纸变量与背景层

| 变量名 | 说明 |
|--------|------|
| `--chat-wallpaper-dark` | 暗色模式聊天背景壁纸路径，如 `'dark.webp'` |
| `--chat-wallpaper-light` | 亮色模式聊天背景壁纸路径，如 `'light.webp'` |

壁纸文件存放于 `public/wallpaper/`，`themeStore` 会在 `fetchThemes()` 时预解析壁纸路径为 `/wallpaper/{basename}.webp` 缩略图 URL，供 `ThemePicker.vue` 预览使用。

### 3.4 Glassmorphism 相关变量

`themes.css` 中定义了 `.glass-panel` 与 `.glass-panel-active` 两个全局类：

```css
.glass-panel {
  background-color: color-mix(in srgb, var(--secondary-bg) 90%, transparent);
  box-shadow: 
    inset 0 1px 0 rgba(255,255,255,0.05),
    0 4px 15px -3px rgba(0,0,0,0.1),
    0 2px 6px -2px rgba(0,0,0,0.05);

```

- 使用 `color-mix()` 实现背景透明化，兼容明暗模式；
- `.glass-panel-active` 添加 `var(--accent-bg)` 与 `var(--highlight-text)` 的淡色叠加，用于选中态。

### 4.1 shortcuts

```ts
shortcuts: [
```

### 4.2 presets

| Preset | 用途 |

## 5. 主题运行时切换

### 5.1 themeStore 的职责

> 文件位置：`src/core/stores/theme.ts`（285 行）

`useThemeStore` 是 Pinia Composition API 风格的 Store，暴露以下状态与操作：

| 导出项 | 类型 | 说明 |
|--------|------|------|
| `mode` | `Ref<ThemeMode>` | 用户选择的模式：`light` / `dark` / `system` |
| `isDarkResolved` | `Ref<boolean>` | 解析后的实际明暗状态（处理 `system` 跟随） |
| `currentTheme` | `Ref<string>` | 当前主题文件名，如 `themes-bear-holiday.css` |
| `currentThemeInfo` | `Ref<ThemeInfo \| null>` | 当前主题的完整元数据与变量 |
| `availableThemes` | `Ref<ThemeInfo[]>` | 全量主题列表（懒加载） |
| `themeThumbnails` | `Ref<Record<string, string>>` | 主题壁纸缩略图 URL 缓存 |
| `setMode(mode)` | `Function` | 切换明暗模式，带 100ms 防抖 |
| `toggleTheme()` | `Function` | 根据 `isDarkResolved` 切换明暗 |
| `applyThemeFile(name)` | `Async Function` | 加载并应用指定主题模块 |
| `initTheme()` | `Async Function` | 启动时初始化：先加载当前主题，空闲时再扫描全量 |

### 5.2 明暗模式解析（light/dark/system）

```
用户选择 mode
    │
    ├──► 'dark'  ────────────────► isDarkResolved = true
    │
    ├──► 'light' ────────────────► isDarkResolved = false
    │
    └──► 'system' ──► matchMedia('(prefers-color-scheme: dark)')
                          │
                          ├──► matches ──► isDarkResolved = true
                          └──► not matches ► isDarkResolved = false
```

- `system` 模式通过 `window.matchMedia` 实时监听系统主题变化；
- 解析结果同步到 `document.documentElement.classList.toggle('dark', ...)` 与 `document.body.classList.toggle('light-theme', ...)`，供组件级 `dark:` 变体使用。

### 5.3 CSS 变量热更新机制

**首次加载**：

```ts
const injectVariables = (vars: Record<string, string>) => {
  // 1. 清除旧主题变量，避免混合态
  for (const key of lastAppliedVarKeys.value) {
    document.documentElement.style.removeProperty(key);
  }
  // 2. 注入新变量
  for (const [key, value] of Object.entries(vars)) {
    document.documentElement.style.setProperty(key, value);
  }
  lastAppliedVarKeys.value = Object.keys(vars);
};
```

**模式切换时重注入**：当 `mode` 从 `dark` 切到 `light`，若当前主题模块已加载，则重新注入对应分支的变量，无需重新读取文件。

**Vite HMR 支持**：

```ts
if (import.meta.hot) {
  setTimeout(() => {
    if (currentTheme.value) applyThemeFile(currentTheme.value);
  }, 100);
}
```

开发环境下修改主题 TS 文件后，Vite 热更新会自动重新执行 `applyThemeFile`，实现样式的实时无刷新生效。

### 5.4 壁纸切换逻辑

壁纸路径存储于主题变量的 `--chat-wallpaper-dark` / `--chat-wallpaper-light` 中，格式如 `'dark.webp'`。`fetchThemes()` 阶段会预解析为缩略图 URL：

```ts
const match = rawPath.match(/url\(['"]?(.*?)['"]?\)/);
let filename = match ? match[1] : rawPath;
filename = filename.split('.')[0] + '.webp';
thumbs[theme.fileName] = `/wallpaper/${filename}`;
```

实际壁纸渲染由聊天页通过 CSS `background-image: var(--chat-wallpaper-dark)` 消费，`ThemePicker.vue` 仅展示缩略图预览。

## 7. 组件与 UnoCSS 的配合

### 7.1 为什么显式使用原子类而非快捷类

设置页原子组件内部**全部显式书写原子类**（如 `flex items-center justify-between py-3.5 px-1`），而非使用 `flex-center` 或 `card` 快捷类。原因如下：

1. **自包含性**：组件样式不依赖外部配置文件，复制/复用时不会因为缺少 `uno.config.ts` 而失效；
2. **可搜索性**：在 IDE 中可以直接搜索 `bg-blue-500` 找到所有使用处，若封装为 `btn` 则搜索失效；
3. **避免隐式依赖**：快捷类的变更会影响所有使用方，而原子类的组合是局部的；
4. **移动端密度**：设置页需要精确到 `py-3.5`、`text-[10px]` 等微调，通用快捷类粒度太粗。

### 7.2 动态类绑定策略

组件通过 `:class="[...]"` 数组绑定实现多态样式：

```vue
<!-- SettingsCard.vue -->
<div :class="[
  variant === 'glass' ? 'bg-white/5 border-white/10' : 'bg-black/5 dark:bg-white/5 ...',
  variant === 'danger' ? 'border-red-500/20 bg-red-500/5' : '',
  'border rounded-2xl shadow-sm',
]">
```

策略要点：
- 固定类放最后，变体条件类放前面，确保覆盖优先级正确；
- 暗色模式通过 `dark:` 变体前缀（`presetUno` 提供）与 `dark` 类名联动，无需手动写媒体查询；
- 颜色相关样式优先使用 CSS 变量（如 `text-primary-text`），布局/间距使用原子类。

### 7.3 主题感知组件的实现

设置页组件通过以下三种方式感知主题：

| 方式 | 示例 | 层级 |
|------|------|------|
| CSS 变量 | `color: var(--primary-text)` | 最优先，动态切换无刷新 |
| `dark:` 变体 | `dark:bg-white/5 dark:hover:bg-white/10` | 明暗模式二元切换 |
| 语义化快捷类 | `bg-primary text-danger` | UnoCSS theme.colors 定义 |

## 8. 与 Rust 后端的 IPC 交互

### 8.1 Tauri Commands 调用表

设置原子组件层**不直接调用** Tauri Commands。所有设置持久化由上层页面组件通过各自的 Store 完成。themeStore 作为例外，需要与后端同步主题状态：

| 操作 | 前端 | 后端 | 方向 |
|------|------|------|------|
| 主题配置持久化 | `localStorage.setItem('vcp-theme-name', ...)` | — | 前端本地 |
| 主题配置读取 | `localStorage.getItem('vcp-theme-name')` | — | 前端本地 |
| 后端推送主题变更 | `listen('onThemeUpdated', ...)` | `emit('onThemeUpdated', fileName)` | Rust → JS |

### 8.2 事件监听表

> 文件位置：`src/core/stores/theme.ts` 第 245–254 行

```ts
const unlistenThemePromise = listen('onThemeUpdated', (event) => {
  const fileName = event.payload as string;
  if (fileName !== currentTheme.value) {
    applyThemeFile(fileName);
  }
});

onScopeDispose(() => {
  unlistenThemePromise.then((fn: UnlistenFn) => fn()).catch(() => {});
});
```

| 事件名 | 方向 | 负载 | 用途 |
|--------|------|------|------|
| `onThemeUpdated` | Rust → JS | `string`（主题文件名） | 后端同步服务或其他模块推送主题变更时，前端自动应用 |

Store 在 `onScopeDispose` 中清理 `matchMedia` 监听器与 Tauri 事件监听器，避免热重载后产生重复监听。

## 9. 设计决策与注意事项

### 9.1 为什么用 UnoCSS 而非 Tailwind

| 维度 | Tailwind | UnoCSS |
|------|----------|--------|
| 构建方式 | PostCSS 插件，扫描生成 CSS | 即时引擎，按需生成，零 CSS 产物 |
| 预设扩展 | 配置复杂，需自定义 tailwind.config.js | `defineConfig` 直接组合 presets |
| 图标方案 | 需安装 @heroicons 等包 | `presetIcons` + CDN 按需加载，零依赖体积 |
| 变体组 | 不支持 | `transformerVariantGroup` 原生支持 `hover:(bg-red text-white)` |
| 属性化 | 不支持 | `presetAttributify` 支持 `<div bg-red>` |

对于移动端应用，UnoCSS 的即时生成特性显著降低了构建产物体积，CDN 图标方案避免了将数千个 SVG 打包进 APK。

### 9.2 为什么 shortcuts 不多于 5 个

快捷类是**全局命名空间**，每增加一个就增加一层隐式约定：

- `btn` 和 `card` 是通用到无法归属任何领域的原语；
- 若添加 `settings-row`、`form-input` 等领域 shortcuts，会导致样式与配置文件的紧耦合；
- 4 个 shortcuts 的规模使新成员可以在 10 秒内全部掌握，心智负担极低。

### 9.3 为什么 themes.css 和 uno.config.ts 双重定义

| 文件 | 定义内容 | 消费者 | 目的 |
|------|----------|--------|------|
| `themes.css` | `--layer-*` CSS 变量 | `<style>` 标签、CSS 文件 | 供纯 CSS 场景（如 `vcp-scrollable` 滚动条样式）使用 |
| `uno.config.ts` | `theme.zIndex` | UnoCSS 引擎 | 生成 `z-content`、`z-drawer` 等原子类 |
| `layers.ts` | `LAYER_*` TS 常量 | TypeScript 逻辑 | 供 JS/TS 动态计算 z-index（如 `getPageZIndex(stackIndex)`） |

三者语义完全一致，只是消费场景不同。修改层级时必须同步更新三处。

### 9.4 为什么显式使用原子类而非过度依赖 shortcuts

这是 VCP Mobile **UI 美学宪法**的一部分（详见 `docs/UI_LAYER_ARCHITECTURE.md`）：

> 技术精确感：ID、UUID、状态码强制使用 Monospace 字体。灰度优先，彩色仅用于状态指示。

原子类的显式堆砌本身就是一种"技术精确感"的表达——开发者可以精确知道一个元素的每一像素来源。快捷类的黑盒封装与这种美学目标相悖。

### 9.5 注意事项

1. **主题变量注入时机**：`initTheme()` 优先只加载当前主题，全量扫描通过 `requestIdleCallback` 延迟执行，避免启动阻塞；
2. ** legacy 主题兼容**：`LEGACY_THEME_MAP` 将旧版中文文件名映射到新的英文文件名，保证老用户升级后主题不丢失；
3. **安全区回退**：`env(safe-area-inset-top, 24px)` 在 CSS 层面提供物理刘海屏回退值；
4. **图标尺寸强控**：`[class^="i-"] { width: 1.2em; height: 1.2em; }` 确保所有 Iconify 图标尺寸统一，不受外部样式影响。

## 10. 术语速查表

| 术语 | 说明 |
|------|------|
| **原子类 (Atomic Class)** | 单一用途的 CSS 工具类，如 `flex`、`bg-red-500`、`py-2` |
| **快捷类 (Shortcut)** | UnoCSS 中通过 `shortcuts` 定义的复合类，如 `btn` = 多个原子类组合 |
| **CSS 变量注入** | 通过 `document.documentElement.style.setProperty('--key', 'val')` 动态修改 `:root` 变量 |
| **HMR (Hot Module Replacement)** | Vite 开发服务器的热更新机制，修改文件后无需刷新页面即可看到变更 |
| **动态导入 (Dynamic Import)** | `import.meta.glob()` 在构建时扫描模块，运行时按需加载 |
| **`color-mix()`** | CSS 原生函数，用于在两种颜色之间按比例混合，本项目用于玻璃态透明背景 |
| **`dark:` 变体** | UnoCSS/Tailwind 的暗色模式前缀，仅在父元素有 `.dark` 类时生效 |
| **Vite Attributify** | `presetAttributify` 提供的语法，允许将原子类写为 HTML 属性而非 class |
| **Iconify CDN** | `presetIcons` 的 CDN 模式，通过 `https://esm.sh/` 按需下载图标数据 |
| **安全区 (Safe Area)** | 移动设备屏幕边缘的非显示区域（刘海、圆角、手势条），通过 `env(safe-area-inset-*)` 适配 |
