# 07. 指令与语义化层级

## 1. 概述

### 1.1 领域定位

`core/directives/` 与 `core/constants/layers.ts` 是 VCP Mobile 前端层的**基础设施**，承担两项正交职责：

1. **全局指令**：通过 Vue Custom Directive 封装移动端高频交互（ intersection 观察、长按触发），将 DOM 操作细节从组件中剥离。
2. **语义化层级**：建立 11 级 Z-Index 体系，通过 CSS 变量 + UnoCSS Theme + TypeScript 常量三重机制，根治"魔法数字"泛滥的层叠失控问题。

两者均不直接参与业务逻辑，而是作为**横切关注点**被全项目引用。

### 1.2 模块构成表

| 文件 | 行数 | 职责 |

| `src/assets/themes.css` | 131 | CSS 变量定义（含层级变量、主题色、玻璃面板、消息气泡样式） |

### 1.4 核心设计原则

| 原则 | 说明 |
|------|------|
| **指令无侵入** | 指令不依赖特定组件状态，通过 CustomEvent 解耦回调 |
| **事件驱动反馈** | `v-longpress` 触发时调用 `navigator.vibrate(50)`，提供物理感知 |
| **语义化 > 数字** | 全局层级禁止 `z-50`、`z-[999]` 等魔法数字，必须使用命名层级 |
| **三层一致性** | 同一语义层级在 CSS、UnoCSS、TypeScript 中数值严格一致 |
| **动态封顶** | 页面栈层级随索引递增，但上限封顶，避免无限侵占上层空间 |

## 2. 全局指令

### 2.1 v-intersection-observer

> 文件位置：`src/core/directives/intersectionObserver.ts`

#### 2.1.1 实现原理

基于浏览器原生 `IntersectionObserver API` 封装。指令在 `mounted` 时为元素创建观察者，在 `unmounted` 时断开连接并清理引用。

```
el 挂载
  │
  ▼
┌──────────────────────────┐
│ new IntersectionObserver │──> threshold 默认为 0.1
│  (callback, options)     │    可传入 binding.value 覆盖
└──────────┬───────────────┘
           │
           ▼
    ┌──────────────┐
    │ isIntersecting│──> Yes: dispatch 'intersect'  CustomEvent
    │              │      No:  dispatch 'unintersect' CustomEvent
    └──────────────┘
           │
    ┌──────┴──────┐
    ▼             ▼
 .modifiers.once  持续监听
 observer.unobserve(el)  保留 observer
```

#### 2.1.2 源码分析

```ts
// src/core/directives/intersectionObserver.ts 第 4–28 行
export const vIntersectionObserver: Directive = {
  mounted(el, binding) {
    const options = binding.value || { threshold: 0.1 };
    const observer = new IntersectionObserver((entries) => {
      entries.forEach((entry) => {
        if (entry.isIntersecting) {
          el.dispatchEvent(new CustomEvent('intersect', { detail: entry }));
          if (binding.modifiers.once) {
            observer.unobserve(el);   // 一次性触发后自动卸载
          }
        } else {
          el.dispatchEvent(new CustomEvent('unintersect', { detail: entry }));
        }
      });
    }, options);

    el._observer = observer;
    observer.observe(el);
  },
  unmounted(el) {
    if (el._observer) {
      el._observer.disconnect();
      delete el._observer;
    }
  },
};
```

| 特性 | 说明 |
|------|------|
| **默认阈值** | `threshold: 0.1`，元素可见 10% 即触发 |
| **事件名** | `intersect` / `unintersect`，携带 `entry` detail |
| **once modifier** | `v-intersection-observer.once` 触发一次后自动取消观察 |
| **内存安全** | `unmounted` 时 `disconnect()` 并删除 `_observer` 属性 |

#### 2.1.3 使用场景

- **无限滚动**：列表底部放置哨兵元素，进入视口时触发下一页加载。
- **懒加载**：图片/头像在接近视口时才开始请求网络资源。
- **消息列表可见性**：`ChatView.vue` 中通过 `@intersect` 控制消息气泡的入场动画触发时机，避免不可见元素消耗渲染资源。

### 2.2 v-longpress

> 文件位置：`src/core/directives/longpress.ts`

#### 2.2.1 实现原理

以 `touchstart`/`mousedown` 启动计时器，`touchend`/`mouseup`/`touchmove`/`mousemove` 取消计时。若 600ms 内无移动/抬手，则触发回调。

```
touchstart / mousedown
  │
  ▼
┌─────────────────────┐
│ setTimeout 600ms    │
│ (pressTimer)        │
└──────────┬──────────┘
           │
     ┌─────┴─────┐
     ▼           ▼
  touchmove   touchend / touchcancel
  mousemove   mouseup / mouseleave
     │           │
     ▼           ▼
  isTouchMoved   clearTimeout
  = true        (pressTimer = null)
     │
     ▼
  超时检查：!isTouchMoved ? executeLongPress() : 无操作
```

#### 2.2.2 源码分析

```ts
// src/core/directives/longpress.ts 第 3–98 行（关键逻辑）
export const vLongpress: Directive = {
  mounted(el: HTMLElement, binding: DirectiveBinding) {
    const callback = binding.value;
    const delay = 600; // 长按触发时间 ms
    let pressTimer: number | null = null;
    let isTouchMoved = false;

    const executeLongPress = (e: Event) => {
      if (navigator.vibrate) {
        navigator.vibrate(50);  // 物理震动反馈
      }
      callback(e);
    };

    const start = (e: Event) => {
      if (e.type === 'mousedown' && (e as MouseEvent).button !== 0) return;
      isTouchMoved = false;
      pressTimer = window.setTimeout(() => {
        if (!isTouchMoved) executeLongPress(e);
      }, delay);
    };

    const cancel = () => { clearTimeout(pressTimer); pressTimer = null; };
    const move = () => { isTouchMoved = true; cancel(); };

    // Touch + Mouse 双端覆盖
    el.addEventListener('touchstart', start, { passive: true });
    el.addEventListener('touchend', cancel);
    el.addEventListener('touchmove', move, { passive: true });
    el.addEventListener('touchcancel', cancel);
    el.addEventListener('mousedown', start);
    el.addEventListener('mouseup', cancel);
    el.addEventListener('mousemove', move);
    el.addEventListener('mouseleave', cancel);

    // 拦截桌面端原生右键菜单，转交长按回调
    const onContextMenu = (e: Event) => {
      e.preventDefault();
      cancel();
      executeLongPress(e);
    };
    el.addEventListener('contextmenu', onContextMenu);

    (el as any)._longpressCleanup = () => { /* 移除全部 8 个监听器 */ };
  },
  unmounted(el: HTMLElement) {
    if ((el as any)._longpressCleanup) (el as any)._longpressCleanup();
  }
};
```

| 特性 | 说明 |
|------|------|
| **触发阈值** | 600ms，兼顾误触容忍与响应速度 |
| **移动检测** | `touchmove` / `mousemove` 即标记 `isTouchMoved = true` 并取消计时 |
| **震动反馈** | 触发时 `navigator.vibrate(50)`，提供移动端物理感知 |
| **桌面端兼容** | `mousedown`/`mouseup` + `contextmenu` 拦截，统一为长按语义 |
| **左键过滤** | `mousedown` 非左键（`button !== 0`）直接忽略 |

#### 2.2.3 使用场景

- **消息气泡长按菜单**：`ChatBubble.vue` 中绑定 `v-longpress`，触发后通过 `overlayStore.openContextMenu()` 展示复制/删除/收藏等操作。
- **Agent 卡片长按**：`AgentList.vue` 中长按 Agent 卡片唤起快速操作面板。
- **桌面端右键统一**：通过 `contextmenu` 拦截，确保 PC 浏览器调试时行为与移动端一致。

### 2.3 指令注册方式（main.ts）

## 3. 语义化 Z-Index 体系

### 3.1 11 级层级总表

项目采用 **11 级分层架构**，每层间隔 10，预留插入空间。

| 层级 | 语义名 | 数值 | 用途 | UnoCSS 类 | CSS 变量 | TS 常量 |
|------|--------|------|------|-----------|----------|---------|
| L0 | `content` | 0 | 页面内容默认层 | `z-content` | `--layer-content` | `LAYER_CONTENT` |
| L1 | `local` | 10 | 页面内局部悬浮（置底按钮、角标） | `z-local` | `--layer-local` | `LAYER_LOCAL` |
| L2 | `drawer` | 20 | 左右抽屉 + 遮罩 | `z-drawer` | `--layer-drawer` | `LAYER_DRAWER` |
| L3 | `overlay` | 30 | 全局覆盖容器 | `z-overlay` | `--layer-overlay` | `LAYER_OVERLAY` |
| L4 | `page` | 40–49 | SlidePage 页面栈 | `z-page` | `--layer-page` | `LAYER_PAGE_BASE` |
| L5 | `sheet` | 50 | BottomSheet、ModelSelector | `z-sheet` | `--layer-sheet` | `LAYER_SHEET` |
| L6 | `dialog` | 60 | Prompt、ContextMenu、UpdatePrompt | `z-dialog` | `--layer-dialog` | `LAYER_DIALOG` |
| L7 | `viewer` | 70 | AttachmentViewer、FullScreenEditor、AvatarCropper | `z-viewer` | `--layer-viewer` | `LAYER_VIEWER` |
| L8 | `editor` | 80 | HtmlPreviewBlock（全屏HTML） | `z-editor` | `--layer-editor` | `LAYER_EDITOR` |
| L9 | `toast` | 90 | Toast 通知 | `z-toast` | `--layer-toast` | `LAYER_TOAST` |
| L10 | `boot` | 100 | BootScreen（启动屏） | `z-boot` | `--layer-boot` | `LAYER_BOOT` |
| L11 | `gate` | 110 | PermissionGate（权限引导页） | `z-gate` | `--layer-gate` | `LAYER_GATE` |

**层叠秩序口诀**：

```
内容 < 局部 < 抽屉 < 覆盖 < 页面 < 弹层 < 对话框 < 查看器 < 编辑器 < Toast < 启动 < 门禁
```

### 3.2 三层保障机制（CSS 变量 + UnoCSS Theme + TS 常量）

同一语义层级通过三套机制同时暴露，确保模板、样式、逻辑代码都能引用同一套语义。

```
└─────────────────────────────────────────────────────┘
```

### 3.3 各层级的代表性组件

| 语义层级 | 代表性组件 | 定位方式 | 备注 |
|----------|-----------|----------|------|
| `content` | 消息气泡、话题列表、Agent 卡片 | `relative` / `static` | 默认层，不额外声明 z-index |
| `local` | `ChatView` 置底按钮、`GroupStopAllButton`、`StreamingTag` | `fixed` / `absolute` | 页面内局部悬浮，不跨页面 |

| `page` | `SettingsView`、`AgentSettingsView`、`SyncSessionView` | `fixed`（SlidePage） | 页面栈动态计算 |
| `sheet` | `BottomSheet`、`ModelSelector` | `fixed` | 底部上滑弹层 |

| `viewer` | `AttachmentViewer`、`FullScreenEditor`、`AvatarCropper` | `fixed` | 全屏查看/编辑 |

| `boot` | `BootScreen` | `fixed` | 启动屏，仅在初始化时存在 |
| `gate` | `PermissionGate` | `fixed` | 权限门禁，仅引导流程显示 |

### 3.4 页面栈动态层级计算

> 文件位置：`src/core/stores/overlay.ts` 第 43–47 行

`overlayStore` 维护一个 `pageStack` 数组，记录当前打开的 SlidePage 序列。

```ts
const getPageZIndex = (type: string) => {
  const index = pageStack.value.findIndex(p => p.type === type);
  if (index === -1) return LAYER_PAGE_BASE;
  return LAYER_PAGE_BASE + Math.min(index, LAYER_PAGE_MAX_OFFSET);
};
```

**行为**：
- 第一个页面 → z-index 40
- 第二个 → 41
- ...
- 第十个及以上 → 49（封顶）

**封顶原因**：Toast(50) 以上有独立的提示/弹层系统，页面栈不应无限侵占上层空间。

### 4.4 壁纸系统与背景层

> 文件位置：`src/App.vue` 第 52–82 行

壁纸路径解析逻辑由 `backgroundStyle` computed 属性实现：

```
themeStore.currentThemeInfo
  │
  ▼
┌──────────────────────────────┐
│ 读取当前主题变量              │
│ --chat-wallpaper-light 或    │
│ --chat-wallpaper-dark        │
└──────────────┬───────────────┘
               │
         ┌─────┴─────┐
         ▼           ▼
     有值          无值 / "none"
         │              │
         ▼              ▼
   提取文件名      尝试另一模式的壁纸
   去掉路径        （暗色 ↔ 亮色 Fallback）
   去掉引号
   强制 .webp
         │
         ▼
   backgroundImage:
     url("/wallpaper/{filename}.webp")
```

**关键处理**：
1. **Fallback 机制**：若当前模式无壁纸，自动尝试另一模式。
2. **路径清洗**：`replace(/^.*[\\\/]/, "")` 剥离路径，仅保留文件名。
3. **强制格式转换**：`split(".")[0] + ".webp"`，统一使用 `public/wallpaper/` 下的 WebP 优化资源。

## 5. 使用规范

### 5.1 何时使用语义化层级

以下场景**必须**使用语义化层级（`z-{name}` / `var(--layer-{name})` / `LAYER_*`）：

- 全局覆盖层组件：`fixed` / `absolute` 定位，可能跨多个页面存在。
- 抽屉、弹窗、Toast、启动屏等用户可见的层叠元素。
- `SlidePage` 页面栈中的任何页面。

### 5.2 何时使用常规数值层级

以下场景**允许**继续使用 `z-10`、`z-20` 等常规数值：

- 组件内部微观层级：角标、hover 覆盖、加载状态、伪元素阴影。
- 处于独立层叠上下文内的局部叠加（如 `TopicList.vue` 角标、`ChatBubble.vue` 装饰边框）。
- 这些层级不影响全局秩序，因为它们被父容器的层叠上下文隔离。

### 5.3 新增覆盖层组件的层级选择决策树

```
新增组件是否为 fixed/absolute 全局覆盖层?
  │
  ├─ No ──► 使用常规数值 z-10/z-20（局部微观层级）
  │
  └─ Yes
       │
       ▼
  能否归入现有 11 个语义层级?
       │
  ├─ Yes ──► 使用对应语义层级（z-drawer / z-dialog 等）
  │
  └─ No
       │
       ▼
  是否有明确的新语义类别?
       │
  ├─ Yes ──► 按间隔 10 新增层级，同步修改三层
  │            layers.ts / uno.config.ts / themes.css
  │            并更新本文档
  │
  └─ No ──► 重新审视组件定位，可能不属于覆盖层
```

## 7. 设计决策与注意事项

### 7.1 为什么使用 UnoCSS 而非 Tailwind？

| 维度 | UnoCSS | Tailwind |
|------|--------|----------|
| **配置灵活性** | `defineConfig` 纯 TS 配置，shortcuts / presets / transformers 自由组合 | 基于 `tailwind.config.js`，扩展需插件 |
| **按需生成** | 基于 AST 的即时原子类生成，零未使用 CSS | JIT 模式类似，但构建链更重 |
| **Icon 集成** | `presetIcons` + CDN 直接内联 SVG，无需单独图标库 | 需 `@heroicons` 等额外依赖 |
| **变体组** | `transformerVariantGroup` 支持 `hover:(bg-blue text-white)` | 需 `tailwind-merge` 辅助 |

VCP Mobile 选择 UnoCSS 的核心原因是**配置即代码**——所有预设、快捷类、主题色均收敛在 `uno.config.ts` 一个文件中，与 Vue/Vite 工具链无缝集成。

### 7.2 为什么需要三层保障机制？

如果只提供一种引用方式，会在某些场景下造成使用障碍：

| 场景 | 可用机制 | 原因 |
|------|----------|------|
| Template 中写 `class="z-dialog"` | UnoCSS | 最简洁 |
| `<style scoped>` 中写 `z-index: var(--layer-dialog)` | CSS 变量 | 避免在 scoped 样式中依赖构建时生成的类名 |
| Store 中动态计算 `getPageZIndex(2)` | TS 常量 | 需要运行时数值运算 |

三层机制共同保证：**无论在哪一层代码中，开发者都能以语义而非数字思考层级问题**。

### 7.3 为什么禁止 `z-[999]` 和 `z-50`？

**血训来源**：重构前（commit `f7dbd21` 之前），项目中存在以下失控现象：

- `BottomSheet`、`ModelSelector` 使用 `z-[999]` / `z-[1000]`
- `HtmlPreviewBlock` 使用 `z-[10000]`
- `SlidePage` 默认 50、`overlayStore` 基准 50、`ChatView` 置底按钮 50，三者语义不同却共享同一数字
- `ContextMenuSheet` 直接挂载在 `GlobalOverlayManager` 内部且未使用 Teleport，导致 `z-[100]` 实际等效于 `z-60`

这些不是"足够高"，而是"放弃思考"。语义化层级通过**命名即约束**，强制开发者在添加组件时明确其层级定位。

### 7.4 为什么 `v-longpress` 阈值设为 600ms？

- **低于 400ms**：容易与点击混淆，误触率高。
- **高于 800ms**：用户感知延迟明显，体验拖沓。
- **600ms**：在"不干扰正常点击"与"快速响应"之间取得平衡，同时与系统原生长按行为（约 500ms）保持接近。

### 7.5 为什么 `v-intersection-observer` 使用 CustomEvent 而非回调参数？

指令的 `binding.value` 通常用于传递配置，若同时传递回调，会导致参数语义混杂。通过 `CustomEvent` 解耦后：

- 配置（threshold、rootMargin）走 `binding.value`
- 触发信号走 DOM 事件 `@intersect` / `@unintersect`
- 组件侧使用更直观，与 Vue 的事件监听风格一致
